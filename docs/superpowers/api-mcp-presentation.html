Lasi un AI extern
sa-ti caute dosarele,
fara sa-i dai cheile.

Un token personal, doar pentru citire, prin care Claude Desktop sau ChatGPT pot interoga dosare, ICCJ si RNPM — exact cat le permiti, nimic mai mult.

3 capabilitati: dosare · iccj · rnpm doar citire default-deny: restul = 403 revocare instant OpenAPI 3.1 auto-descriere web mode · desktop zero-impact

Stare: implementat. Backend + frontend livrate TDD pe branch feat/api-mcp-piesa-a (16 taskuri, ~1850 teste verzi, 5 runde de review adversarial). Macheta de mai jos ramane o simulare in browser a fluxului — nu apeleaza serverul real — dar reflecta comportamentul construit.

02

Parcurge fluxul

Patru pasi, in ordine. Tokenul pe care-l creezi la pasul 1 te urmareste pana la pasul 4 — il pui in config, intrebi AI-ul, apoi vezi urma in audit.

Creezi un token in Setari → „Acces API”

Ii dai un nume, alegi ce poate citi, optional o expirare si un plafon de captcha. Secretul apare o singura data.

Il pui in configul Claude Desktop

Puntea MCP locala (piesa B) primeste tokenul prin variabila de mediu si vorbeste cu API-ul tau.

  claude_desktop_config.json

Tokenul circula doar prin HTTPS, e redactat din loguri, si — fiind in antetul Authorization: Bearer — e imun la CSRF (un browser nu-l poate atasa cross-origin).

Intrebi AI-ul ceva

Apasa o intrebare. Urmaresti cererea trecand prin poarta, apoi vezi auto-load-more: la dosare AI-ul aduce automat paginile (cu badge exact-match si calitatea fiecarei parti — reclamant/parat/inculpat); la RNPM cheltuie automat captcha pana la plafon. Intrebarile de AI/monitorizare sunt respinse demonstrativ.

04

Plasele de siguranta

Pentru ca un agent AI da rafale, nu click-uri, si un token poate sa nu expire — controalele de mai jos sunt implementate si intarite prin 5 runde de review (2× panel multi-model + agenti + audit adversarial complet).

🚧

Default-deny pe segment

Tokenul ajunge doar pe rutele allowlistate explicit. Match pe granita de segment, ca /api/dosare sa nu deschida din greseala /api/dosare-iccj; encoding-ul ambigu (%2f/..) e respins inainte de matching.

Fix CRITICAL din review
📖

Doar citire, pe metoda

Permis pe metoda HTTP: GET + un singur POST de cautare (RNPM), exact-match ca sa nu prinda sub-rute. Orice scriere sub aceleasi prefixe → refuzata.

HIGH
🛡️

Bearer = fara CSRF

Tokenul se accepta DOAR din antetul Authorization: Bearer, niciodata din cookie. Bypass-ul de origine se aplica doar dupa autentificare reusita cu token valid.

HIGH
🚫

Fara cache · HTTPS-only

Cache-Control: no-store pe orice raspuns de token (secretul one-time la creare nu ajunge in cache-uri). In productie tokenul cere HTTPS, altfel 426.

review
👤

Management doar sesiune

Crearea / revocarea tokenurilor e session-only si trece prin verificarea de origine (anti-CSRF pe mutatii); un token nu poate administra tokenuri.

Fix HIGH runda 4
⏱️

Limita de rata per token

Fiecare token are propriul plafon, mai strans decat al unei sesiuni umane, aplicat inaintea portii (si rafalele pe rute interzise sunt numarate). Env-ul e clampat (fara valori toxice).

MEDIU
💸

Plafon captcha atomic

Optional per token, cu rezervare atomica (BEGIN IMMEDIATE, fara race) si fail-closed pe valori invalide. RNPM costa bani reali — un token scurs nu depaseste plafonul.

decizie ta
📜

Auto load-more

Serverul da un bloc + total/hasMore; AI-ul continua singur. Dosare: auto pana la pragul de fan-out, apoi un singur prompt, cu exactMatch + calitate parte. RNPM: auto pana la plafon.

decizie ta
🔌

Siguranta ICCJ

Un intrerupator global cu numaratoare ponderata pe clasa de apelant (token = 0.25, plafonat sub prag): un token in bucla nu mai poate pica ICCJ pentru UI si monitorizare. 503 cu Retry-After aliniat la cooldown.

HIGH
🧾

Audit + alerta IP nou

Folosire (inclusiv 403/429) + creare + revocare, cu IP proxy-aware. Prima folosire dintr-un IP nou (doar reusite) → email prin infra existenta.

decizie ta
🗺️

Auto-descriere OpenAPI

GET /api/v1/openapi.json descrie rutele, scopes si formele de raspuns; reachable de un token (inaintea portii), fara sa expuna nimic in plus.

nou
🔴

Revocare instanta

Validare in DB la fiecare cerere, fara cache. Plus buton „revoca toate” (cu confirmare). Singurul buton de panica de care ai nevoie.

confirmat OK
05

Ce s-a decis

Scopes
3: dosare (incl. termene), iccj, rnpm
Expirare
Optionala in UI — implicit fara, dar poti alege 30/90/365 zile
Acces
Default-deny + doar citire — restul aplicatiei e 403 pentru token
Export Excel/PDF
Exclus din token; doar date brute, cu pagina plafonata server-side
Captcha RNPM
Nelimitat din start + plafon optional per token (alegerea ta, contra recomandarii panelului)
Load-more
Dosare: auto pana la prag + 1 prompt, cu exact-match si calitate parte. RNPM: complet automat pana la plafon, fara confirmare (debitor/creditor)
ICCJ
Intrerupator pe clasa de apelant + limita per token, fara cap global de rata
Audit
Da — jurnal append-only + alerta la IP nou + buton revoca-tot
